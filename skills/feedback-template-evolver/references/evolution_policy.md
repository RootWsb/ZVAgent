# Feedback Evolution Policy

## Promote Immediately

Promote to a proposal when feedback includes:

- explicit future language: "以后", "下次", "默认", "总是", "记住"
- a reusable structure: "这个结构不错", "保留这个模板"
- repeated length/depth correction
- domain-specific output preferences

## Keep As Log Only

Do not apply as a permanent rule when:

- the feedback is ambiguous
- the user is reacting to a one-off answer
- the feedback conflicts with safety, source, or financial-risk rules
- a newer explicit instruction supersedes it

## Apply Rules

- Prefer appending a single bullet.
- Keep bullets concrete and testable.
- Avoid broad personality rewrites.
- Preserve domain boundaries.
- Newer user requests override older accepted rules.

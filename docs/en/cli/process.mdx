title: Process Management
description: Manage ZVAgent process lifecycle with zv commands
---

Process management commands control the ZVAgent background process. These commands are only available in the terminal.

## start

Start the ZVAgent service. Runs as a background daemon by default and automatically tails logs.

```bash
zv start
```

**Options:**

| Option | Description |
| --- | --- |
| `-f`, `--foreground` | Run in foreground, not as a background daemon |
| `--no-logs` | Don't tail logs after starting |

## stop

Stop the running ZVAgent service.

```bash
zv stop
```

## restart

Restart the ZVAgent service (stop then start).

```bash
zv restart
```

**Options:**

| Option | Description |
| --- | --- |
| `--no-logs` | Don't tail logs after restart |

## update

Update code and restart the service. Automatically performs:

1. Pull latest code (`git pull`)
2. Stop current service
3. Update Python dependencies
4. Reinstall CLI
5. Start service

```bash
zv update
```

<Warning>
  If `git pull` fails (e.g., uncommitted local changes), the update aborts and the service remains unaffected.
</Warning>

## status

Check ZVAgent service status, including process info, version, and current model/channel configuration.

```bash
zv status
```

## logs

View service logs.

```bash
zv logs
```

**Options:**

| Option | Description | Default |
| --- | --- | --- |
| `-f`, `--follow` | Continuously tail log output | No |
| `-n`, `--lines` | Show last N lines | 50 |

Examples:

```bash
# View last 100 lines
zv logs -n 100

# Continuously tail logs
zv logs -f
```

## install-browser

Install Playwright and Chromium browser for the [browser tool](/en/tools/browser).

```bash
zv install-browser
```

<Tip>
  Only needed when using browser tools (web browsing, screenshots, etc.).
</Tip>

## run.sh Compatibility

If ZV CLI is not installed, you can use `run.sh` to manage the service:

| zv command | run.sh equivalent |
| --- | --- |
| `zv start` | `./run.sh start` |
| `zv stop` | `./run.sh stop` |
| `zv restart` | `./run.sh restart` |
| `zv update` | `./run.sh update` |
| `zv status` | `./run.sh status` |
| `zv logs` | `./run.sh logs` |

<Note>
  The `zv` command is recommended — it provides cleaner syntax and richer features. It is automatically installed via the one-click install script.
</Note>
